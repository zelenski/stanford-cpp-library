41 /*
42  * Class: DawgLexicon
43  * ------------------
44  * This class is used to represent a <b><i>lexicon,</i></b> or word list.
45  * The main difference between a lexicon and a dictionary is that
46  * a lexicon does not provide any mechanism for storing definitions;
47  * the lexicon contains only words, with no associated information.
48  * It is therefore similar to a set of strings, but with a more
49  * space-efficient internal representation. The <code>DawgLexicon</code>
50  * class supports efficient lookup operations for words and prefixes.
51  *
52  * Currently the recommended implementation of a lexicon is the
53  * <code>Lexicon</code> class, which supports more operations than this one.
54  *
55  * <p>As an example of the use of the <code>DawgLexicon</code> class, the
56  * following program lists all the two-letter words in the lexicon
57  * stored in <code>EnglishWords.dat</code>:
58  *
59  *<pre>
60  * int main() {
61  * DawgLexicon english("EnglishWords.dat");
62  * for (string word : english) {
63  * if (word.length() == 2) {
64  * cout << word << endl;
65  * }
66  * }
67  * return 0;
68  * }
69  *</pre>
70  */
